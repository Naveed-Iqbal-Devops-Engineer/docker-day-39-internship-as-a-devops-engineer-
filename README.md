Docker uses a client-server architecture:
Docker Engine: Complete platform installed on your Linux system to build and run containers.
Docker Client: CLI that communicates with Docker Daemon.
Docker CLI: Command-line tool that users run to interact with Docker via terminal.
Docker Daemon: Background service responsible for building, running, and managing containers.
Docker Image: Read-only template used to create containers.
Docker Container: Running instance of an image.
Docker Hub: Public registry to store and retrieve Docker images.
Docker Volume: Mechanism for persistent data storage outside the container filesystem.
